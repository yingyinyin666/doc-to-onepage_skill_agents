# doc-to-onepage

**将"复杂信息"转化为"清晰决策表达"的AI能力层**

---

## 一句话定义

将任意文档转化为**"可决策的一页纸（OnePage）"**，并生成可直接用于汇报的网页与飞书文档。

**帮你把"信息"变成"说服力"。**

---

## 核心价值

本技能不是简单的格式转换工具，而是一个**汇报内容重构引擎**，提供三大核心能力：

### 1. 内容重构（Content Reframing）
- 从"信息堆积" → "结论优先"
- 自动提炼：标题、核心结论、关键要点、数据支撑

### 2. 信息压缩（High-Density Compression）
- 将长文压缩为：**1页**可读完、**3分钟**可讲清
- 保留关键决策信息，删除冗余内容

### 3. 汇报表达增强（Presentation Enhancement）
- 汇报开场句（Opening）
- 结论总结句（Summary）
- Q&A预判（关键问题提示）

# 📐 OnePage 通用设计规范（Design System）

所有 OnePage 必须遵循统一规范，确保一致性和可决策性。

---

## 1️⃣ 页面结构规范（必须遵守）

所有 OnePage 必须遵循统一骨架：

```
[Header - 结论区]
[Core - 核心逻辑区（最多3块）]
[Execution - 执行区（可选）]
[Risk - 风险区（建议必有）]
[Appendix - 附录区（折叠）]
```

---

## 2️⃣ 信息密度规范（核心约束）

**📏 硬性规则：**
- 总阅读时长：≤ 3分钟
- 核心模块数：≤ 3
- 每模块要点：≤ 3条
- 每条字数：≤ 40字

**👉 超过就必须折叠**

---

## 3️⃣ 结论区规范（最高优先级）

**必须包含：**
- 标题（Title）
- 一句话结论（Conclusion）
- 行动建议（CTA）

**✨ 标准写法：**
```
【标题】XX项目优化方案
【结论】建议立即推进XX方案
【CTA】是否批准资源投入？
```

**👉 禁止：**
- 没有结论
- 结论模糊（如"建议优化"）

---

## 4️⃣ 模块结构规范（卡片化）

每个模块必须是：

```
模块标题（Why / What / Impact）
↓
卡片（1~3个）
  - 一句话结论
  - 数据（如有）
```

---

## 5️⃣ 视觉层级规范（强约束）

**字号层级：**

| 层级 | 用途 |
|------|------|
| H1 | 标题 |
| H2 | 模块标题 |
| H3 | 卡片标题 |
| Body | 正文 |

**👉 禁止所有文字同一大小**

**留白规范：**
- 模块间距：≥ 48px
- 卡片间距：≥ 16px

**颜色规范：**
- 只允许3类颜色：主色（品牌）、中性色（灰）、强调色（CTA/结论）
- **👉 禁止彩虹色**

---

## 6️⃣ 数据展示规范

**使用条件：**
只有当数据能回答"所以呢？"时才展示

**图表规则：**
- ≤ 2个图表
- 必须配结论文字

---

## 7️⃣ CTA规范（强制）

**每个 OnePage 必须有：**

👉 **一个明确动作：**
- 是否批准？
- 是否推进？
- 是否分配资源？

---

## 8️⃣ 网页输出质量原则（必须遵守）

> 来源：真实用户反馈提炼，适用于所有风格的网页生成。

### A. 标题层级三原则
1. **H1 主标题**：核心主题，支持拆分为主标题+副标题（`meta.subtitle`），禁止出现无意义英文如「ONEPAGE REPORT」
2. **H2 模块标题**：字号明显大于正文，与正文之间留白 ≥ 80px，可配专属语义图标
3. **H3 子项标题**：结论前置，把核心收益/要求放在句首，让用户扫一眼即可 get 重点

### B. 结论区视觉权重最高
- 结论区（KEY TAKEAWAY）必须是页面第一个视觉焦点
- 浓缩三要素：**必做要求 + 核心激励/收益 + 违规/风险后果**
- 使用更高对比度背景、更大内边距，与普通卡片明显区分

### C. 对比数据必须可视化，禁止纯文本列表
- 阶梯型数据（如激励梯度）→ 用 2 列极简表格或进度条，数字单独高亮
- 并列对比（如合规/违规示范）→ 用 2×2 卡片网格
- 枚举规则（如 4 条合规原则）→ 用图标卡片网格，不用 `✅` 前缀纯文本

### D. 功能色语义规范（所有深色风格通用）
| 颜色 | 语义 | 典型场景 |
|------|------|---------|
| 绿色 | 正向 / 合规 | 激励权益、合规操作 |
| 橙色 | 警示 / 注意 | 风险提示、限制条件 |
| 红色 | 禁止 / 负面 | 违规后果、强制要求 |
| 蓝色 | 操作 / 跳转 | 使用说明、操作路径 |
| 黄色 | 高亮关键词 | 数字、法规名称、核心短语 |

### E. 高亮节制原则
- **只高亮关键词/数字/法规名称**，禁止整行或整段高亮
- 高亮覆盖率控制在正文的 10-15%，超出则失去引导作用
- 每个子项 `<strong>` 不超过 3 处

### F. 深色背景可读性规范
- 行高：≥ 1.75（深色背景行间距需比浅色背景更大）
- 正文字色：≥ `#d0e8d0`（对比度 ≥ 4.5:1，满足 WCAG AA）
- 禁止细字重（font-weight ≤ 300）在深色背景上使用
- 段落间距：每个子项之间 ≥ 12px，避免内容挤成一团

### G. 子项分组用卡片，禁止用细横线
- 细横线在深色背景下几乎不可见，**用轻透明背景卡片做区隔**
- 每个子项单独成卡：背景 + 内边距 + 圆角，清晰分隔
- 卡片内容超过 3 行时必须增大内边距，保证呼吸感

### H. 操作路径步骤化
- 操作指引禁止纯文本箭头（如「A → B → C」）
- 改用流程图组件（`flow` 类型 section）或带编号的步骤卡片
- 核心操作按钮/入口用功能蓝色高亮标注，与普通路径文字区分

### I. 移动端字号下限（强制）
- 正文最小字号：≥ 12px（`0.75rem`）
- 标题最小字号：≥ 16px（`1rem`）
- 左右安全边距：≥ 14px，避免文字贴屏幕边缘

### J. meta 字段支持（所有模板）
`data.json` 中可配置 `meta` 对象自定义页面元信息：
```json
{
  "meta": {
    "header_tag": "平台官方公告",
    "subtitle": "适用范围：度假旅游服务类目",
    "footer_text": "© 抖音生活服务平台 · 度假业务"
  }
}
```

### K. 多列版式自动选择（子项级）
子项（sub-item）数量决定 grid 布局，禁止永远竖向堆叠：

| 子项数 | 版式 | CSS class |
|--------|------|-----------|
| 1 | 全宽（默认竖排） | `sub-stack` |
| 2 | 两列等宽 | `sub-grid sub-cols-2` |
| 3 | 三列等宽 | `sub-grid sub-cols-3` |
| 4+ | 两列流式（2×N 等高网格） | `sub-grid sub-cols-2` |

**核心规则**：
- 同行格子必须 **等高**（`align-items: stretch`），格子内用 `flex-direction: column` + `.sub-content { flex: 1 }` 撑满
- 禁止出现"孤儿半格"（如 bento 第 4 项只占半行）
- 平板（≤ 900px）：3 列退为 2 列，末尾奇数项跨满
- 移动端（≤ 640px）：所有网格退为单列

### L. 卡片级双列并排（章节级）
当相邻两个章节都是"轻量级"时，自动用 `.card-pair` 容器并排渲染：
- **轻量判定**：子项数 ≤ 3 **且** 所有子项文本总字数 ≤ 600
- 两张并排卡片使用 `align-items: stretch` 确保等高
- 移动端退为单列
- **效果**：打破"永远全宽"的单调节奏，页面出现宽+窄交替的视觉变化

### M. 3D 倾斜与光泽效果
- 对 dark / light / corporate / warm / folder 风格启用鼠标悬停 3D 倾斜（`perspective(1000px) rotateX/Y ±4deg`）
- 每张卡片注入 `.card-shine` 覆层，悬停时产生跟随鼠标的径向光泽
- 物理隐喻风格（retro / scrapbook / dossier）和特殊风格（cartoon / blackboard）不启用 3D 倾斜

### N. 内联进度条与图标系统
- 正文中 `<strong>数字%</strong>` 自动追加 56px 宽的内联进度条（`.inline-progress`），数值驱动宽度
- 子项标题自动匹配中文业务关键词（目标→🎯、保障→🛡️、赔付→💸、应急→🚨 等 40+ 映射），渲染为 emoji 前缀
- Metric 数字卡片保留 0→实际值 的滚动数字动画

## 🎨 9. 卡通宣传册型 (cartoon-brochure)

**🎯 目标：** 面向消费者/用户的服务或产品宣传，活泼、亲切、易读

**适用场景：**
- 服务流程介绍（如旅游、教育、医疗服务）
- 产品功能宣传
- 活动/大促推广说明
- 面向 ToC 用户的内容分发

**🧱 结构：**
```
服务标题（Title - 醒目大字）
↓
服务流程（Flow - 图文步骤）
↓
核心亮点（Highlights - 2×2彩色卡片）
↓
服务承诺（Promise - 保障说明）
↓
行动入口（CTA - 扫码/联系方式）
```

**⚙️ 特殊规则：**
- 必须有：清晰的步骤流程（≤5步）
- 必须有：彩色分区（绿/黄/紫/橙轮替）
- 必须有：一个明确的 CTA（扫码 / 咨询 / 立即预约）
- 避免：过度文字堆砌，每卡片 ≤ 40字
- 避免：商务深色背景，保持明亮活泼

**🎨 视觉风格：** `cartoon`
- 背景：天空蓝渐变（蓝→浅黄）
- 卡片：白色圆角卡片，顶部彩色色条（绿/黄/紫/橙）
- 字体：粗体，活泼有力
- 标题：白色大字，天蓝渐变头部
- 配色：明亮鲜艳，禁止灰暗沉闷色调

## 📋 10. 黑板报通知型 (blackboard-notice)

**🎯 目标：** 发布公告、通知、公示，信息传达清晰权威

**适用场景：**
- 重要通知 / 紧急公告
- 放假 / 停课 / 停工安排
- 规章制度 / 须知说明
- 活动安排 / 日程公示
- 温馨提示 / 告知事项

**🧱 结构：**
```
公告标题（Title - 醒目粉笔大字）
↓
核心内容（Content - 要点清单）
↓
具体安排（Detail - 时间/地点/人员）
↓
注意事项（Notice - 强调提醒）
↓
落款 / 生效时间（Footer - 发布方 + 日期）
```

**⚙️ 特殊规则：**
- 必须有：标题明确指出通知类型（公告/通知/公示）
- 必须有：生效时间或发布日期
- 必须有：发布方署名
- 避免：口语化表达，保持正式庄重
- 避免：信息过度堆砌，每条要点 ≤ 30字

**🎨 视觉风格：** `blackboard`
- 背景：深墨绿黑板纹理，四角暗角效果
- 文字：粉笔白色，微量闪烁动效模拟真实粉笔
- 标题栏：白色虚线花边框，顶部「★ 公 告 栏 ★」标签（黄色）
- 卡片：透明底 + 粉笔白描边，左侧彩色粉笔竖线（红/黄/蓝/绿轮替）
- 重点内容：黄色高亮强调（`<strong>` 文字变金黄色）
- 结论/CTA：红色警示框，醒目但不失庄重

## 交互流程（Agent 必须遵循）

### 1️⃣ 启动条件
当用户表达以下意图时触发：
- "生成 onepage"
- "写一页纸报告"
- "帮我总结成汇报"

### 2️⃣ 模板推荐（必须执行）
- AI分析文档内容
- 自动推荐最佳模板
- 用户确认或修改

### 3️⃣ 信息补充（轻量对话）
- 目标受众（管理层 / 跨团队）
- 语气风格（专业 / 简洁 / 轻松）
- 强调重点（可选）
- 图表需求（可选）

### 4️⃣ 文档处理
- 自动识别格式
- 提取文本内容

### 5️⃣ 内容分析
- 结构识别
- 关键数据提取
- 逻辑关系重建
- **设计规范校验**

### 6️⃣ 生成初稿
```bash
python3 scripts/generate_onepage.py \
  --template <模板名> \
  --title "<标题>" \
  --refs "<参考文档>" \
  --output "./output/onepage_generated.lark.md"
```

### 7️⃣ 创建飞书文档
```bash
python3 scripts/create_lark_doc.py create \
  --title "<标题>" \
  --content "$(cat ./output/onepage_generated.lark.md)"
```

### 8️⃣ 交互式优化（循环）
- 接收用户反馈
- 调用 `apply_updates.py` 更新

### 9️⃣ 生成网页版本
```bash
python3 scripts/export_onepage_json.py \
  --onepage ./output/onepage_generated.lark.md \
  --template <模板名>

python3 scripts/build_web_onepage.py \
  --data ./output/data.json \
  --style <风格>
```

### 🔟 最终交付
- ✅ 飞书文档链接
- ✅ Web 页面链接
- ✅ 本地文件路径
