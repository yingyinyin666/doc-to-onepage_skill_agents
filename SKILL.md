# doc-to-onepage

**将"复杂信息"转化为"清晰决策表达"的AI能力层**

## 一句话定义

将任意文档转化为**"可决策的一页纸（OnePage）"**，并生成可直接用于汇报的网页与飞书文档。

**帮你把"信息"变成"说服力"。**

## 核心价值

本技能不是简单的格式转换工具，而是一个**汇报内容重构引擎**，提供三大核心能力：

- **内容重构**：从"信息堆积" → "结论优先"
- **信息压缩**：将长文压缩为1页可读完、3分钟可讲清
- **汇报增强**：汇报开场句、结论总结句、Q&A预判

## 使用方式

当用户提供文档（飞书链接、本地文件路径）时，自动执行以下流程：

```
1. 解析文档（多格式支持）
2. AI推荐最优模板
3. 生成OnePage初稿  ← Claude 在此步骤主动填充内容
4. 创建飞书文档
5. 支持多轮修改
6. 生成网页版本
7. 🔄 自检与自修复（确保质量）
```

### 步骤3「生成OnePage初稿」的执行规范

调用 `generate_onepage.py` 后，Claude 必须**主动读取输出文件**，并根据参考文档内容逐章节填充，而不是直接交付空模板。

填充规则：
- **结论放首屏**：第一个 `## 结论与建议` 章节必须包含核心观点+CTA决策问题，网页会渲染为渐变横幅
- **章节标题编号**：从 `## 01 xxx` 开始编号，依次递增。编号格式为两位数字+空格+标题，如 `## 01 背景`、`## 02 方案`
- **章节标题要带结论**：如 `## 01 背景：抖音旅行管家已落后竞品5年`，标题本身传递信息，不只是分类标签
- **每节不超过3个要点**：列表/子标题控制在3条以内，避免信息过载
- **数据优先**：若参考文档包含数字、指标，优先引用到正文中
- **对比内容用表格**：方案对比、选项对比一律用 Markdown 表格
- **结尾必须有CTA**：末尾章节（决策建议/后续行动）必须包含明确的决策问题，如"是否批准推进？"
- **风险不可省略**：若参考文档未提及风险，Claude 需根据方案特点主动补充1-2条潜在风险

### 章节灵活性规则

模板只是起点，**章节必须根据原文档实际内容灵活调整**：

- **可增**：原文有重要内容但模板未覆盖时，新增章节（如原文有详细的实验设计，可新增"实验方案"章节）
- **可删**：模板中的章节在原文中找不到对应内容时，直接删除，不要硬凑空内容
- **可合并**：两个模板章节在原文中高度相关时，合并为一个（如"资源"和"排期"合并为"资源与排期"）
- **可拆分**：原文某部分内容特别丰富时，可拆分为多个章节
- **总量控制**：最终章节数（不含结论）控制在 4-6 个，超过 6 个说明信息没有充分压缩

### 风格选择

生成网页前，向用户确认视觉风格（若用户未指定，默认 `dark`）：

| 风格 | 适用场景 | 视觉特点 |
|------|----------|----------|
| `dark` | 高层汇报、产品评审 | 暗色背景+毛玻璃卡片+彩色光晕，高级感强 |
| `light` | 日常周报、团队分享 | 白色背景+淡灰卡片+蓝色强调，清爽易读 |
| `corporate` | 正式决策、跨部门汇报 | 深蓝顶部+白色内容区+严谨排版，专业权威 |
| `warm` | 团队介绍、项目复盘 | 米色背景+暖棕色调+柔和阴影，亲和温暖 |

使用方式：`python3 scripts/build_web_onepage.py --data data.json --style dark --outdir output/web`

## OnePage设计规范

### 页面结构
```
[Header - 结论区] → [Core - 核心逻辑区] → [Risk - 风险区] → [Appendix - 附录]
```

### 动效规范
- **时长**：200-400ms
- **Fade In**：内容默认出现
- **Stagger**：卡片依次出现（80ms间隔）
- **Hover Lift**：卡片悬浮效果

## 🔄 自检与自修复系统

在生成网页后，自动进行"结构 + 视觉"的质量检查，并进行自我优化，确保最终结果符合汇报标准。

### 核心流程
```
生成HTML → 自动检查 → 发现问题 → 修复优化 → 输出最终版本
```

### 校验标准

| 检查项 | 说明 | 权重 |
|--------|------|------|
| 结论优先 | 首屏可见结论区域 | 20分 |
| CTA行动号召 | 有明确的决策问题 | 20分 |
| 风险说明 | 包含风险与依赖 | 15分 |
| 模块数量 | 核心模块≤3个 | 15分 |
| 视觉层级 | 清晰的字号/颜色区分 | 15分 |
| 内容长度 | 无信息过载 | 15分 |

### 自动修复能力

脚本会自动修复以下问题：
- 缺少结论区域 → 自动添加结论横幅
- 缺少风险模块 → 自动添加风险说明
- 缺少CTA → 自动添加决策问题

### 使用方式
```bash
python3 scripts/validate_onepage.py \
  --html output/web/index.html \
  --auto-fix \
  --output output/web/index.html
```

### 产品价值
**AI不仅帮你写页面，还会帮你"审稿 + 改稿"，直到它像一份合格的汇报材料。**
